## What Needs to Be Built

### Route restructure

Current `/` is marketplace. Landing page moves in at `/`, marketplace shifts to `/marketplace`.

```
/ (new)                   → Landing page (split hero)
/marketplace (new)        → Move current / marketplace here + add NL map
/banks/[id]               → unchanged
/banks/[id]/buy           → unchanged
/banks/[id]/buy/confirmed → unchanged
/banks/[id]/report        → unchanged
/methodology              → unchanged
/dashboard/corporate      → NEW
/dashboard/foodbank       → NEW
```

### 1. Landing page (`/`)

Top → bottom:

1. **Navbar** — "Climate Harvest" logo, links: Marketplace | Methodology | For Food Banks
2. **HeroSplit** — two columns, `bg-emerald-900` gradient, white text:
   - Left (corporate): "Turn food rescue into audit-ready CSR proof" → "Browse packages →" (`/marketplace`)
   - Right (foodbank operator): "Unlock corporate funding for your rescue operations" → "Join as food bank →" (`/dashboard/foodbank`)
3. **ImpactStats bar** — 3 stats computed from `banks.json` aggregate:
   - Total annual tCO2e across 6 banks
   - Total annual kg rescued
   - Total households weekly
4. **HowItWorks** — 3-step horizontal: Food bank uploads annual data → FRAME engine computes CO2e → Corporate buys package, gets quarterly CSR report
5. **Map teaser** — small Leaflet map, 6 pins (hardcoded lat/lng), "Explore all food banks →" → `/marketplace`
6. **Footer** — FRAME attribution, methodology link

### 2. NL Map on Marketplace (`/marketplace`)

Add above card grid. Use `react-leaflet`:
- 6 pins from `banks.json`, one per bank
- Click pin → scroll to / highlight that bank's card
- Popup: bank name, annual tCO2e, "View →" link
- NL bounding box

Add coordinates to `banks.json` or inject in `lib/banks.ts`:

| eindhoven | 51.4416 | 5.4697 |
| groningen | 53.2194 | 6.5665 |
| breda | 51.5719 | 4.7683 |

### 3. Corporate Dashboard (`/dashboard/corporate`)

Post-purchase view. Mock state: Heineken N.V., Rotterdam package, Q2 2026.

1. **Header** — "Heineken N.V." · "Corporate sponsor" badge · "View report" link
2. **Summary cards** (4 shadcn `Card`):
   - `600 tCO2e avoided` · `323,389 kg rescued` · `€25,000 invested` · `Q2 2026 · report ready`
3. **ImpactChart** — Recharts `BarChart`, category breakdown from Rotterdam `category_mix` × `emission_factors`:
   - produce, dry_goods, dairy, bakery, meat, prepared, eggs — emerald bars
4. **Attribution panel** — "7.77% of Rotterdam's annual baseline" · shadcn `Progress` bar
5. **Report section** — ESRS E1 + S3 badge, "View full report →" links to `/banks/rotterdam/report`
6. **Subscription row** — `€25,000` · status `paid` (green shadcn `Badge`) · Q2 2026

### 4. Food Bank Dashboard (`/dashboard/foodbank`)

Operator view. Mock state: Voedselbank Amsterdam, no sponsor yet.

1. **Header** — "Voedselbank Amsterdam" · "Food bank operator" badge · "Noord-Holland" region
2. **Funding cards** (3 shadcn `Card`):
   - `€0 received` · `0 active sponsors` · `Annual data: not uploaded`
3. **UploadWidget** — drag-and-drop zone (PDF / Excel / CSV):
   - Year selector (2024 / 2025)
   - On file drop → mock progress: Uploading (1s) → Parsing with Claude (2s) → Computing FRAME (1s) → Done ✓
   - After done: reveal extracted stats from Amsterdam's `banks.json` entry
   - Footer: `ingestion_model: claude-sonnet-4-6`
4. **Impact metrics** (revealed post-upload):
   - `annual_kg_rescued`, `households_weekly`, `people_served`
   - Category donut (Recharts `PieChart`) from Amsterdam `category_mix`
5. **Sponsorship tracker** — empty state: "Your bank will appear in the marketplace once data is uploaded and reviewed"
6. **Source legend** — `extracted` (from PDF) vs `inferred` (national average) labels
