# User Stories & Workflows

**Date:** 2026-04-26
**Status:** Reference doc for finalist pitch + frontend QA
**Scope:** Two primary actors — Corporate sustainability lead, Foodbank operator

---

## Persona 1 — Corporate sustainability lead

**Name (illustrative):** Sanne — Head of Sustainability at a mid-cap NL company (Heineken / Unilever / Philips class)
**Trigger:** CSRD deadline. Annual report needs ESRS E1 (climate) + S3 (affected communities) disclosures. Internal carbon offsets program is under audit pressure — needs verifiable avoided-emissions provenance.
**Constraint:** Solo decision-maker on small purchases (<€100k) — needs to act in one session, no procurement loop.
**Context of use:** Tuesday afternoon, desktop, office. Calm. 30 minutes blocked.

### User stories

| # | As a corporate lead, I want to… | So that… |
|---|---|---|
| C1 | Browse impact funds at a glance | I can size the spend before committing |
| C2 | See the methodology before I trust the number | An auditor can defend it without my help |
| C3 | Drill into one fund's bank-by-bank allocation | I know exactly where my money lands |
| C4 | Buy at a fixed €/tCO₂e price, no negotiation | Procurement skips the RFP loop |
| C5 | Watch the CSR report stream in real time | I get end-of-quarter disclosure copy-paste-ready |
| C6 | Cite every figure back to a source | I survive the Big-4 audit |

### Workflow — happy path

```
/                          Landing — "verified avoided emissions, audit-ready"
   ↓ Browse funds
/marketplace               4 funds × cards, NL province heatmap, filters
   ↓ Click a fund card
/funds/<id>                Fund profile — projected allocation table, historical
                           timeline, "Invest in this fund →"
   ↓ Click CTA
/pricing                   3 tiers: Starter €15k · Partner €25k · Enterprise €50k
   ↓ Click [Buy <Fund Name> →]
/funds/<id>/buy            Solvimon-style mock checkout
   ↓ Confirm payment
/funds/<id>/buy/confirmed  "Your subscription is live" + Generate report CTA
   ↓ Click "View Q2 2026 report" / dashboard
/dashboard/corporate       4 disclosure-grade stats (typographic row, no cards),
                           subscription runway timeline, geographic coverage,
                           provenance scorecard, cost-effectiveness gauge
   ↓ Click "View CSR report →"
/reports/<subId>           SSE-streamed CSRD/ESRS E1+S3 markdown report,
                           generated by Claude with source citations
```

**Time-to-value:** Marketplace → live subscription in **~3 minutes**. Report generation: 20–40 s streaming.

### What happens at each step

**Landing** — Hero claim + CSRD positioning. Province heatmap shows network density. Stats: avoided / yr, rescued / yr, households / wk, food banks. Pitch split: "For corporates" / "For food banks".

**Marketplace** — 4 fund cards (Carbon Certified, Community First, CSRD Complete, Impact Balanced) each scoped by `impact_profile` (`co2_focus`, `social_focus`, `balanced`). Each card shows region, top-N banks count, claim in tCO₂e. Province heatmap below — dark-emerald province fills weighted by foodbank count. Visible province→count list for keyboard / screen-reader users.

**Fund profile** — Hero: fund name + impact profile. Stats trio: Total avoided / Allocated to / Per-bank avg. **Projected allocation table** — every bank in the fund with `weight_pct`, `attributed_kg`, `attributed_tco2e`, `attributed_eur`, computed at purchase time. Historical performance line chart aggregated across the top-N banks. "What you receive" panel: ESRS E1+S3 disclosure, FRAME workings, source citations, provenance ledger. Methodology footer (FRAME, NL counterfactual). Sticky right-rail: single CTA "Invest in this fund →" → `/pricing`.

**Pricing** — Three tiers. Each tier maps to a fund (`profile`):
- Starter €15k/quarter → Social Reach Fund (`social_focus`)
- Partner €25k/quarter → CO₂ Impact Fund (`co2_focus`) — Most popular
- Enterprise €50k/quarter → Balanced Impact Fund (`balanced`)
Click [Buy Fund →] = `BuyTierButton` resolves the right `package.id` and POSTs to `/checkout`.

**Checkout** — Mock Solvimon screen. Confirm → backend creates `FundSubscription(status=paid)`. Allocation engine snapshots the `Allocation[]` rows immediately so the disclosure is locked at purchase time.

**Confirmation** — "Subscription live." CTA: View dashboard / Generate Q2 report.

**Corporate dashboard** — 4-stat typographic row with dividers, no card chrome. Methodology disclosure inline. Pacing bar (quarters realised / contracted). Equivalents (cars, flights, NL-household-years). Subscription runway timeline (8 quarters realised + forecast). Province-weight heatmap of the buyer's allocation. Provenance scorecard (extracted vs inferred %). Cost-effectiveness gauge (€/tCO₂e vs benchmark). Allocation table.

**Report** — `/reports/<subId>` triggers `services/report.py::stream_report`. Claude Sonnet 4.6 streams markdown via SSE. Sections: Executive Summary, ESRS E1-6 table, Methodology Appendix, Allocation Detail, Provenance Ledger. Saved to `data/reports/<subId>.md` for download.

### Edge cases

- **Unauthenticated buy** — `BuyTierButton` returns 401 → redirect `/login?next=/funds/<id>`
- **Subscription failed payment** — `status=failed`, dashboard shows banner with retry
- **Bank capacity sold-through** — package routes to next-best banks; allocation shows new weights
- **Report stream fails** — partial markdown saved; UI shows "Resume from quarter X"

## Persona 2 — Foodbank operator

**Name (illustrative):** Marieke — Operations lead at a regional Voedselbank (Voedselbank Rotterdam-class)
**Trigger:** Just published the annual report (Dutch PDF, 30–80 pages). Wants to attract corporate sponsorship without changing operations or hiring a sustainability consultant.
**Constraint:** Volunteer time. Five minutes per week, max. No technical extraction skill.
**Context of use:** Wednesday morning, desktop in the warehouse office. Phone interruptions. Forms must tolerate her stepping away mid-task.

### User stories

| # | As a foodbank operator, I want to… | So that… |
|---|---|---|
| F1 | Drop my annual report PDF and walk away | I don't lose 4 hours filling forms |
| F2 | See exactly what Claude extracted | I can correct anything wrong before it goes public |
| F3 | Know what's "extracted" vs "inferred" | An auditor (or VBN) can trust the data |
| F4 | Appear in the marketplace once data is in | Corporates can find and sponsor me |
| F5 | See incoming sponsorships and €s | Board reporting takes 10 minutes, not 4 hours |
| F6 | Edit a wrong number without breaking provenance | I keep the audit trail intact |

### Workflow — happy path

```
/                          Landing — "Join as a food bank →" CTA
   ↓
/register                  Email + password + bank name (org_name)
   ↓ Auto-login
/dashboard/foodbank        Empty state: "Drop a PDF to begin"
   ↓ Drag PDF onto UploadWidget
   POST /preprocessing/ingest
   (5 parallel Claude Haiku extractions: volumes, categories, demographics,
    operations, donations — each tagged with provenance source + method)
   ↓ ~30–60 s later
/dashboard/foodbank        Reveal mode — extracted data with provenance badges:
                           - extracted (green)  : taken directly from the PDF
                           - inferred (yellow)  : national-avg / category split
                           - computed (blue)    : FRAME math
   ↓ Edit any field
/dashboard/foodbank/edit   Inline editor — overwriting a value flips its source
                           to "manual" and logs the previous value in
                           ProvenanceRecord history
   ↓ Once happy, click "Publish profile"
/foodbanks/<slug>          Public transparency profile. Indexed by marketplace.
                           - Hero: bank name, region, kg/yr, tCO₂e/yr, households/wk
                           - Category mix donut
                           - Provenance ledger table (every measurement, source,
                             method, date)
                           - Source PDF link (kept verbatim)
   ↓ Wait for sponsorships
/dashboard/foodbank        Sponsorship feed — when a corporate buys a fund whose
                           allocation includes this bank:
                           - notification card "Heineken sponsored €4,200 via
                             CSRD Complete fund (this bank's allocation: 17%)"
                           - quarterly disbursement summary
   ↓ Annual cycle
/dashboard/foodbank        Upload next year's PDF — re-runs extraction, new
                           AnnualReport row, year-over-year delta visible.
```

**Time-to-value:** New foodbank → public profile + marketplace listing in **~5 minutes** (mostly waiting for extraction).

### What happens at each step

**Register** — Standard email/password + org_name. Role auto-set to `foodbank`. Session cookie issued. Redirect to `/dashboard/foodbank`.

**Foodbank dashboard (empty)** — Single hero CTA: drop PDF. Eyebrow text: "extracted by Claude". Below: 3-step explainer (Drop → Claude reads → Each field tagged with provenance). Methodology link.

**Upload** — `UploadWidget` accepts a PDF. POSTs multipart to `POST /preprocessing/ingest`. Backend kicks off the ingestion CLI in-process: 5 parallel Claude Haiku calls, each owning one section of the report. Provenance auto-tagged per field — source ∈ {`extracted`, `inferred_national_avg`, `inferred_category_split`, `inferred_calculation`, `manual`}; method = direct citation or formula reference.

**Foodbank dashboard (revealed)** — Once extraction completes, the dashboard shows:
- Stat ribbon: Annual kg rescued / Annual tCO₂e (FRAME) / Households served weekly
- **Provenance badge ribbon**: % extracted / inferred / computed — confidence band high/medium/low
- Section panels: Food Volume, Food Categories, People Served, Operations, Donations — each row carries an inline `<Badge variant="emerald|warning|outline">` showing source. Click any value to edit (changes source → `manual`, prior value preserved in `ProvenanceRecord` history).
- Year selector — once multiple `AnnualReport` rows exist, switch year and see delta.

**Public profile** (`/foodbanks/<slug>`) — Same data presented for **corporate browsers and auditors**. Read-only. Source PDF link is first-class. Provenance ledger is a sortable table — auditor can cite a specific row directly. **No login required**.

**Sponsorship feed (foodbank dashboard)** — Backend computes `Allocation` rows linking `FundSubscription → Foodbank`. The dashboard reads these in reverse: "for this bank, in this period, who sponsored what".
- Card list: Sponsor org name · fund name · period · €amount · weight in fund
- Total per quarter — what to expect in the bank account
- Disbursement state: Pending / Paid (manual mark for now; future: Solvimon webhook)

**Renewal cycle** — Each year the operator drops the new PDF. The system creates a new `AnnualReport` linked to the same `Foodbank`. `FrameResult` is computed for that year. Year-over-year sparkline appears on the public profile and dashboard.

### Edge cases

- **PDF extraction fails** — Show partial results + manual entry form. Don't block publication.
- **PDF in an unusual format (image-only scan, non-Dutch)** — UI prompts: "Extraction confidence low. Want a human reviewer?" Manual queue (post-launch).
- **Field overwritten by mistake** — Provenance history allows revert. Audit trail intact.
- **No `region` matches the 6 KK regions** — Manual select; falls back to province lookup via lat/lng.
- **Two operators on one foodbank** — Same `org_name`; second registration prompts to join existing org (not built; user permissions deferred).

## Cross-cutting workflows

### A — A corporate finds a *specific* bank

```
/marketplace → "Want a transparency profile for one specific bank?"
section → /foodbanks/<slug>
```

The marketplace footer always offers this escape hatch. Corporates with a pre-existing relationship to (e.g.) Voedselbank Rotterdam can land directly on that bank's profile, then go back to `/marketplace` to find a fund whose allocation includes Rotterdam (the fund-detail page shows projected allocation per bank — corporates can verify their preferred bank is included before buying).

### B — An auditor verifies a published claim

```
Corporate's CSRD report cites: "412 tCO₂e avoided via Climate Harvest
CSRD Complete Fund, Q2 2026, €25,000"
   ↓ Auditor clicks the platform footnote
/reports/<subId>           Read the full Claude-generated disclosure
   ↓ Allocation table links to per-bank profiles
/foodbanks/<slug>          Public profile with full provenance ledger
   ↓ Provenance row links to source PDF
voedselbankrotterdam.nl/jaarverslag-2025.pdf
```

Three clicks, public URLs, no login. **The auditability is the product.**

### C — A foodbank reports back to its board / VBN

```
/dashboard/foodbank        Quarterly snapshot — sponsorships received,
                           projected end-of-year totals, % of annual
                           operating budget covered.
   ↓ Click "Export quarterly statement"
                           PDF: cover page, sponsor list, disbursement
                           schedule, provenance manifest. (Future: SEPA
                           file for the bookkeeper.)
```

## Open questions

1. **Sponsorship disbursement** — manual marking now. Solvimon webhook integration is the natural v2.
2. **Multi-operator org** — current registration assumes 1 user per foodbank. Real-world ops have 2-3.
3. **Multi-year history weighting** — should the FRAME baseline use the latest year only, or a 3-year mean? Spec needs an explicit answer before publication.
4. **Bank capacity caps** — UI doesn't yet show "remaining capacity". Critical for the "isn't this over-selling?" jury question (see `docs/context/finalist-pitch.md`).
