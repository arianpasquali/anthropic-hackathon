Reading the diagram

  • Solid arrows = explicit <Link> / router.push in source.
  • Dashed arrows = conditional redirects (auth gates, ?next= handoff, downloads).
  • Dynamic routes wrap their param in […]; the sitemap fans these out per row in listFoodbanks() / listPackages().

Auth gating

  • /dashboard/* and /reports/[subId] redirect to /login?next=… on 401.
  • /funds/[id]/buy redirects to /login?next=/funds/[id] if the session is missing when submitting.
  • Header shows /dashboard/foodbank link unconditionally — actual access is enforced server-side.
