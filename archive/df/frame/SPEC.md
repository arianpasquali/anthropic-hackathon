# `frame/` — FRAME-aligned attribution engine

## Purpose

Deterministic Python module that turns a foodbank + sponsor amount + quarter into:
- a fully traceable tCO₂e attribution under a methodology aligned with the Global Foodbanking Network's FRAME framework,
- a structured payload for the Claude report-generation prompt,
- a `banks.json` artifact for the marketplace UI.

No LLM lives in this module. Claude is invoked only at the report layer, which receives pre-computed numbers and composes audit-grade narrative around them. This separation is the platform's trust contract: every figure on a report can be re-derived from `frame/` without calling an LLM.

## File layout

```
frame/
├── __init__.py        # module declaration
├── factors.py         # emission factors + NL counterfactual + package economics
├── banks.py           # the 6 demo Dutch foodbanks with provenance
├── calculator.py      # deterministic FRAME computation + payload serializer
├── report.py          # Claude prompt construction (system + user message)
├── run_report.py      # end-to-end runner (calls Claude if API key set)
├── export.py          # banks.json exporter for the Next.js marketplace
├── demo.py            # CLI: prints calculator output for Rotterdam €25k
└── sample_payload.json # last computed payload (regenerated by demo.py)
```

## Public API per module

### `factors.py`

| Symbol | Type | What it is |
|---|---|---|
| `EMISSION_FACTORS` | `dict[str, EmissionFactor]` | 7 food categories, kg CO₂e/kg + source + notes |
| `COUNTERFACTUAL_FACTOR_NL` | `float` (0.93) | NL incineration-with-energy-recovery adjustment |
| `COUNTERFACTUAL_SOURCE` | `str` | Citation for the counterfactual |
| `PACKAGE_PRICE_EUR` | `float` (25,000) | Locked package price |
| `PACKAGE_TCO2E` | `float` (600) | Locked package tCO₂e |
| `PRICE_PER_TCO2E` | `float` (41.67) | Derived |
| `factor_table_markdown()` | function | Pre-rendered factor table for the report appendix |

**Categories:** `produce`, `bakery`, `dairy`, `meat`, `dry_goods`, `prepared`, `eggs`. Non-food items distributed by foodbanks are out of scope for FRAME.

### `banks.py`

| Symbol | Type | What it is |
|---|---|---|
| `Bank` | frozen dataclass | Per-bank operational record |
| `DEFAULT_CATEGORY_MIX` | `dict[str, float]` | Fallback mix derived from VBN Schijf van Vijf framing + Haaglanden's published parcel breakdown |
| `BANKS` | `dict[str, Bank]` | The 6 demo banks |
| `get(bank_id)` / `all_banks()` | functions | Lookup helpers |

`Bank` fields: `id`, `name`, `region`, `annual_kg_rescued`, `category_mix`, `households_weekly`, `people_served`, `is_regional_distribution_centre`, `rdc_satellite_count`, `cluster_banks`, `source_url`, `provenance`.

### `calculator.py`

| Symbol | Type | What it is |
|---|---|---|
| `FrameCalculation` | frozen dataclass | Full result + audit trail |
| `compute(bank, sponsor_amount_eur, quarter, corporate_name)` | function | Core attribution computation |
| `to_report_payload(calc)` | function | Serializer for the report prompt |

`FrameCalculation` fields include `weighted_emission_factor`, `bank_annual_tco2e`, `attributed_tco2e`, `attributed_kg_food`, `attribution_share`, `category_breakdown_tco2e`, `category_breakdown_kg`, `methodology_trail`, `sources`.

### `report.py`

| Symbol | Type | What it is |
|---|---|---|
| `SYSTEM_PROMPT` | `str` | Audit-grade reporter prompt — strict no-hallucination contract |
| `build_user_message(calc)` | function | Constructs the user message with JSON payload + factor table |
| `generate(calc, model, max_tokens)` | function | Calls Claude (requires `ANTHROPIC_API_KEY`) |
| `dry_run(calc)` | function | Returns (system, user) without calling the API |

### `run_report.py` / `demo.py` / `export.py`

CLI entry points. Run from the repo root:

```bash
.venv/bin/python -m frame.demo       # FRAME math + sample payload
.venv/bin/python -m frame.export     # writes web/public/banks.json
.venv/bin/python -m frame.run_report # generates report (or writes prompt files)
```

## Data contracts

### Category mix invariant

Bank's `category_mix` must sum to 1.0 over the food categories listed in `EMISSION_FACTORS`. The calculator validates this on every call (`abs(sum - 1.0) < 0.001`) and raises `ValueError` otherwise. Non-food items must not appear in `category_mix`.

### Attribution math (immutable)

```
weighted_ef       = Σ(share × EF[cat]) × COUNTERFACTUAL_FACTOR_NL
bank_annual_tco2e = annual_kg_rescued × weighted_ef / 1000
attributed_tco2e  = sponsor_amount_eur ÷ PRICE_PER_TCO2E
attributed_kg     = attributed_tco2e × 1000 ÷ weighted_ef
attribution_share = attributed_tco2e ÷ bank_annual_tco2e
```

The package price is locked. Changes to `PACKAGE_PRICE_EUR` or `PACKAGE_TCO2E` invalidate every previously generated report — they must trigger a new methodology version, not be silently changed.

### Report payload schema

`to_report_payload()` returns a JSON-serialisable dict with keys:

- `corporate_name`, `quarter`
- `foodbank` — name, region, households_weekly, people_served (nullable), annual_kg_rescued, is_rdc, rdc_satellite_count, cluster_banks
- `package` — amount_eur, price_per_tco2e (rounded to 2 dp)
- `impact` — attributed_tco2e, attributed_kg_food, attribution_share_pct, bank_annual_tco2e, weighted_emission_factor
- `category_breakdown` — per-category tco2e, kg_food, emission_factor
- `methodology_trail` — list of human-readable steps (verbatim in report appendix)
- `sources` — list of citations

## How to extend

### Add a new bank

1. Source the bank's most recent published annual report (place PDF and extracted .txt in `data/`)
2. Extract: annual food kg (or units; convert at 0.6 kg/unit), households weekly, parcel composition if published, RDC role, source URL
3. Add a `Bank(...)` entry to `BANKS` in `banks.py` with a complete `provenance` string
4. Validate the category mix sums to 1.0 (calculator will raise if not)
5. Re-run `python -m frame.demo` to confirm the new bank appears in the cross-check
6. Re-run `python -m frame.export` to refresh `banks.json` for the marketplace

### Add a new emission factor

1. Add an `EmissionFactor(...)` entry to `EMISSION_FACTORS` in `factors.py` with citable source
2. Update existing banks' `category_mix` to include the new category (must still sum to 1.0)
3. Re-run demo + export

### Change the package price

This is a breaking change. Treat it as a methodology version bump:
- Update `PACKAGE_PRICE_EUR` and/or `PACKAGE_TCO2E`
- Add a methodology version field to `factors.py`
- Old reports retain their version; new reports use the new version
- Document the change in `architecture.md` / `methodology.md`

## Provenance discipline

The platform's trust property is that every figure on a report traces to:
1. A foodbank's published annual report (cited in `Bank.source_url` + `Bank.provenance`), or
2. A peer-reviewed / government emission-factor source (cited in `EmissionFactor.source`), or
3. The deterministic attribution math above.

Estimates derived from operational data (e.g. unit-to-kg conversion at 0.6, or households × parcel-size × weeks for banks that don't publish kg) must be documented in the bank's `provenance` field. The Claude report prompt explicitly forbids inventing numbers — all figures arrive in the payload pre-computed.

## Testing strategy

No formal test suite in the hackathon build. Validation is via `python -m frame.demo`, which:
- Runs the full computation for Rotterdam at €25k
- Cross-checks all 6 banks at €25k (annual tCO₂e + attribution share table)
- Writes `sample_payload.json`

Before extending the module, run `demo.py` and confirm the cross-check table matches your expected values.
